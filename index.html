J'ai déployé mon propre agent IA en production en une soirée. Voilà exactement comment j'ai fait 🦞

Article • OpenClaw + Gemini + WhatsApp + Notion

Il y a quelques jours, j'avais une idée simple : un bot WhatsApp capable de lire et écrire dans mes pages Notion, alimenté par une IA.

Résultat ? SL2S-Bot tourne en production sur un VPS depuis ce soir. Et j'ai tout documenté pour toi.

🛠️ La stack technique (sans prise de tête)

J'ai utilisé OpenClaw, une plateforme open-source d'agents IA basée sur Node.js. Le principe : un fichier JSON central qui connecte tout ensemble.

• 🤖 Modèle : Google Gemini 2.0 Flash Lite (gratuit via AI Studio)
• 📱 Canal : WhatsApp via le skill wacli
• 📝 Mémoire : Notion via intégration API
• 🖥️ Hébergement : VPS Linux, accès SSH

🚀 Les 3 commandes qui changent tout

Une fois connecté en SSH, l'installation se résume à ça :

curl -sSL https://openclaw.io/install.sh | bash
openclaw configure
systemctl --user enable openclaw-gateway.service

Trois commandes. Le bot tourne 24h/24, même PC éteint.

🛡️ Le point qui m'a le plus surpris : la sécurité

OpenClaw intègre un mode "Bunker" que je recommande à 100% :

• elevated: false → l'IA n'a pas les droits admin sur ton serveur
• sandbox: docker → chaque exécution de code est isolée dans un conteneur
• auth.token → l'interface web est protégée par mot de passe

Le principe du Moindre Privilège appliqué à l'IA. Simple et efficace.

🔐 Sécurité Avancée : Le vrai mode "Bunker"

L'installation de base est rapide, mais pour un serveur en production, le document officiel recommande 4 couches de protection supplémentaires :

1. Verrouiller le fichier des secrets (Permissions Linux)
Ton fichier de configuration contient tes clés d'API (Notion, Google). Il est impératif de le protéger physiquement au niveau du système pour que seul ton utilisateur puisse le lire :
chmod 600 ~/.openclaw/openclaw.json

2. Le "System Prompt" restrictif
Il faut brider le comportement de l'IA directement dans sa mémoire. Voici le prompt recommandé pour éviter les catastrophes :
"Tu es un assistant de programmation. Tes actions sont limitées au dossier de travail actuel. Tu ne dois JAMAIS : Sortir du dossier racine du projet (cd.. est interdit). Lire les fichiers cachés (commençant par un point.). Accéder au dossier .ssh ou aux fichiers .env. Exécuter des commandes de suppression massive (rm -rf)."

3. Isoler l'accès Web avec Tailscale
Plutôt que d'exposer l'interface de ton bot sur l'internet public, tu peux la lier à un réseau privé virtuel comme Tailscale. Lance la Gateway avec cette commande :
openclaw gateway --bind tailnet
L'accès se fera alors via une IP locale sécurisée, du type http://100.x.x.x:11111.

4. L'alternative pour le 24h/24 : Screen
Si tu préfères garder un œil sur le terminal en permanence (au lieu d'utiliser systemctl en arrière-plan), tu peux utiliser un terminal virtuel persistant :
• Créer la session : screen -S mon-ia
• Quitter sans couper le bot (Détacher) : Ctrl + A, puis D
• Y retourner plus tard : screen -r mon-ia

⚠️ Les 4 bugs que j'ai rencontrés (et leurs solutions)

Bug 1 — L'IA écrit du Python au lieu de répondre
Fix : désactiver l'interpréteur dans le JSON + profil "messaging".

Bug 2 — openclaw doctor efface ta config
Fix : toujours sauvegarder openclaw.json avant. Toujours.

Bug 3 — Notion ne voit aucune page malgré la clé API
Fix : sur Notion → page → ... → Connect to → sélectionner ton intégration. Étape invisible mais bloquante.

Bug 4 — Une virgule oubliée bloque tout
Fix : jq . /root/.openclaw/openclaw.json avant chaque redémarrage.

✅ Ma checklist avant de fermer le terminal

□ openclaw status affiche tout en vert
□ QR Code WhatsApp scanné
□ Intégration Notion autorisée sur au moins une page
□ Sauvegarde du JSON sur mon PC
□ openclaw security audit sans erreur critique

💡 Ce que j'en retiens

Déployer un agent IA en production n'est plus réservé aux équipes avec 3 ingénieurs DevOps. Avec les bons outils, une soirée suffit.

Ce qui prend du temps, c'est comprendre les subtilités (le JSON, les permissions Notion, le mode Bunker). Mais une fois documenté, tout devient reproductible.
